| Token | CSS Variable | Value | Usage |
|-------|-------------|-------|-------|
| xs | `--space-1` | 4px | Icon gaps, badge padding |
| sm | `--space-2` | 8px | Inline element gaps, button vertical padding |
| sm+ | `--space-3` | 12px | Compact list spacing |
| md | `--space-4` | 16px | Default element spacing, button horizontal padding |
| md+ | `--space-5` | 20px | Input padding, card internal gap |
| lg | `--space-6` | 24px | Card padding, section heading margin |
| xl | `--space-8` | 32px | Layout column gaps |
| 2xl | `--space-12` | 48px | Section vertical padding (mobile) |
| 3xl | `--space-16` | 64px | Section vertical padding (desktop) |
| 4xl | `--space-20` | 80px | Hero vertical padding |
| 5xl | `--space-24` | 96px | Page-level section breaks |

Exceptions:
- Header pill: `padding: 10px 12px` (10px not on scale — preserved from existing spec per CONTEXT.md D-05; acceptable as a one-off header exception)

- Header positioner: `top: 18px` — preserved from existing spec (not a spacing scale value; a layout offset)

## Typography

### Font Stack

```css
--font-sans: "Inter", ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
--font-mono: "JetBrains Mono", "Fira Code", "Cascadia Code", ui-monospace, "Courier New", monospace;
```

`font-optical-sizing: auto` on `body`. `-webkit-font-smoothing: antialiased` and `-moz-osx-font-smoothing: grayscale` on `body`.

### Type Scale

Fluid sizes using `clamp()` — no breakpoint-scoped `font-size` overrides. All values locked in CONTEXT.md D-09 and D-10.

| Role | CSS Size | Weight | Line Height | Letter Spacing | Token |
|------|----------|--------|-------------|----------------|-------|
| Display / H1 | `clamp(2.5rem, 5vw + 1rem, 5rem)` | 700 | 1.02 | `var(--tracking-tight)` = `-0.04em` | heading |
| H2 | `clamp(1.75rem, 3vw + 0.75rem, 3rem)` | 700 | 1.06 | `var(--tracking-heading)` = `-0.03em` | heading |
| H3 | `clamp(1.25rem, 1.5vw + 0.5rem, 1.75rem)` | 600 | 1.3 | `var(--tracking-heading)` = `-0.03em` | heading |
| Body large | `1.125rem` (18px) | 400 | 1.65 | none | body |
| Body default | `1rem` (16px) | 400 | 1.6 | none | body |
| Body small | `0.875rem` (14px) | 400 | 1.55 | none | body |
| Label / eyebrow | `0.75rem` (12px) | 600 | 1.4 | `0.08em` (positive, uppercase) | label |

**Weights in use: 400 (regular) and 600–700 (semibold/bold) only.**

### Letter-Spacing Tokens

```css
--tracking-tight:   -0.04em;   /* h1 only */
--tracking-heading: -0.03em;   /* h2, h3 */
--tracking-label:    0.08em;   /* eyebrow / label text — uppercase only */
```

## Color

### 60 / 30 / 10 Split

| Role | Value | CSS Token | Usage |
|------|-------|-----------|-------|
| Dominant (60%) | `#0a0a0b` | `--color-bg` | Page background — near-black, not pure black |
| Secondary (30%) | `#111113` / `#18181b` / `#232326` | `--color-surface-1/2/3` | Section backgrounds, cards, elevated states |
| Accent (10%) | `#7c5cfc` | `--color-accent` | **Reserved for: "Book a call" button fill, nav active indicator dot, active state on any interactive element, canvas particle/line color, cursor-glow spotlight tint** |
| Destructive | `#f87171` | `--color-error` | Form validation error states only — not used for UI actions in Phase 1 |

### Full Token Block (two-layer: primitives → semantic)

**Primitives (raw values, never used directly in component CSS):**

```css
/* Violet scale */
--violet-600:  #7c5cfc;
--violet-400:  #9b7fff;
--violet-dim:  rgba(124, 92, 252, 0.12);   /* spotlight glow background */

/* Zinc / grey scale */
--gray-950:    #0a0a0b;
--gray-900:    #111113;
--gray-800:    #18181b;
--gray-750:    #232326;

/* Signal */
--cyan-400:    #22d3ee;
--green-400:   #4ade80;
--amber-400:   #fbbf24;
--red-400:     #f87171;
```

**Semantic tokens (used in all component CSS):**

```css
/* Backgrounds */
--color-bg:         var(--gray-950);       /* page background */
--color-surface-1:  var(--gray-900);       /* section backgrounds */
--color-surface-2:  var(--gray-800);       /* card backgrounds */
--color-surface-3:  var(--gray-750);       /* elevated cards, hover states */

/* Borders */
--color-border:        rgba(255,255,255,0.08);   /* default border */
--color-border-strong: rgba(255,255,255,0.14);   /* header border, card border on hover */
--color-border-hover:  rgba(255,255,255,0.16);   /* hovered border */
--color-border-focus:  rgba(255,255,255,0.28);   /* focus/active border */

/* Text — all verified >4.5:1 against --color-bg */
--color-text-primary:   #f4f4f5;                   /* headings, nav labels — ~17:1 */
--color-text-secondary: #a1a1aa;                   /* body copy, descriptions — ~7:1 */
--color-text-muted:     rgba(255,255,255,0.50);    /* captions, metadata — decorative only, not body text */

/* Accent */
--color-accent:         var(--violet-600);
--color-accent-dim:     var(--violet-dim);
--color-accent-hover:   var(--violet-400);

/* Signal */
--color-error:          var(--red-400);
--color-success:        var(--green-400);
--color-cyan:           var(--cyan-400);           /* data/technical indicators only */

/* Header glassmorphism */
--header-bg-default:    rgba(8, 10, 10, 0.78);
--header-bg-scrolled:   rgba(8, 10, 10, 0.92);
```

**Accent reserved for — explicit list:**
1. "Book a call" button background fill
2. Nav active-section indicator
3. Canvas particle color (`rgba(124, 92, 252, 0.6)`) and connection line color
4. Cursor-glow spotlight on cards (`rgba(124, 92, 252, 0.12)`)
5. Hover state on the "Book a call" button (`--color-accent-hover`)

**Accent is NOT used on:** body text, section borders, secondary nav links, background gradients, decorative dividers.

## Motion Tokens

```css
--duration-fast: 150ms;
--duration-base: 250ms;
--ease-out:      cubic-bezier(0.16, 1, 0.3, 1);   /* Linear spring easing */
```

**Rules:**
- All CSS `transition` declarations use `var(--duration-base) var(--ease-out)` unless specified otherwise
- `--duration-fast` used for micro-interactions: button hover color change, border shimmer
- Never use `ease-in-out` for entrance or hover transitions
- All canvas and CSS animation must be gated with `prefers-reduced-motion` check

**Source:** CONTEXT.md D-03; STACK.md Motion Principles

## Component Specifications

### Header

**Structure:** Two-element split — outer positioner + inner visual panel.

```
.header-positioner   — position: fixed; top: 18px; left: 50%; transform: translateX(-50%);
                        z-index: 50; width: min(1120px, calc(100% - 32px));
                        NO backdrop-filter here (iOS Safari constraint)

.site-header         — backdrop-filter: blur(18px) saturate(180%);
                        -webkit-backdrop-filter: blur(18px) saturate(180%);
                        background: var(--header-bg-default);
                        border: 1px solid var(--color-border-strong);
                        border-radius: var(--radius-md);
                        padding: 10px 12px;
                        display: flex; align-items: center; justify-content: space-between;
                        transition: background var(--duration-base) var(--ease-out);
```

**Scrolled state:** JS adds `.scrolled` class to `.header-positioner` at 30px scroll offset. `.scrolled .site-header` transitions background to `var(--header-bg-scrolled)`.

**CRITICAL constraint:** `transform` and `backdrop-filter` MUST NEVER be on the same element. The wrapper/inner split is non-negotiable for iOS Safari correctness. (PERF-02 gate depends on this.)

**Source:** CONTEXT.md D-05, D-06; CLAUDE.md Key Design Decisions; REQUIREMENTS.md HEAD-01

## iOS Safari Constraints

These are hard constraints — violations WILL cause rendering bugs on iOS Safari.

1. `backdrop-filter` and `transform` MUST NOT be on the same element (split wrapper pattern required)
2. `backdrop-filter` must always include `-webkit-backdrop-filter` prefix
3. Scroll listener must use `{ passive: true }` flag
4. `canvas` init must be deferred via `requestIdleCallback` (fallback: `setTimeout(fn, 100)`)
5. rAF loop must pause on `visibilitychange` (document.hidden check)
6. resize handler on canvas must be debounced at 150ms

**Source:** CONTEXT.md D-05; REQUIREMENTS.md HEAD-01, PERF-02; PITFALLS.md T1, T4, T5, T6, PR4

## CSS File Architecture

Top-to-bottom order in `styles.css` (mirrors DOM):

```
1.  Design tokens — primitives block, then semantic block (all :root vars)
2.  Reset / base — *, body, html, img, a, button
3.  Typography scale — h1–h3, p, .eyebrow, line-height, font stack
4.  Shared utilities — .btn-primary, .btn-secondary, .reveal, .section-heading
5.  Header — .header-positioner, .site-header, .site-nav, .nav-link, .btn-book-call
6.  [sections follow in later phases, in DOM order]
14. Animation keyframes
15. Media queries — 940px breakpoint (consolidated block at bottom)
16. Media queries — 640px breakpoint (consolidated block at bottom)
17. prefers-reduced-motion overrides
```

**Rule:** No scattered inline breakpoints inside component rules. Media queries live at the bottom in two consolidated blocks only.

**Source:** ARCHITECTURE.md CSS Architecture

## JS Architecture (Phase 1 scope)

Phase 1 introduces or modifies:

1. **Header scroll handler** — listens on `window` with `{ passive: true }`; toggles `.scrolled` on `.header-positioner` at 30px offset; `data-header` attribute moves to `.header-positioner` (not `.site-header`)
2. **Nav toggle** — existing `data-nav-toggle` / `data-nav` pattern preserved; CSS transition for collapse/expand is tasteful slide + fade
3. **Active nav tracking** — new `IntersectionObserver` watches `<section id="*">` elements; adds `.is-active` to matching `.nav-link`; `rootMargin: '-40% 0px -55% 0px'`

All querySelector calls guarded: `const el = document.querySelector('[data-x]'); if (!el) return;`

**Source:** CONTEXT.md code_context section; ARCHITECTURE.md JavaScript Architecture
