## Typography

All values are from the established Phase 1 token system. No new type styles are introduced.

| Role | Size | Weight | Line Height | Letter Spacing | Token / Rule |
|------|------|--------|-------------|----------------|--------------|
| Body / paragraph | 1rem (16px) | 400 | 1.5 | 0 | body rule |
| Secondary body | 1.12rem (~18px) | 400 | 1.5 | 0 | `.section-heading p`, `.contact-copy p` |
| Heading h3 | `clamp(1.25rem, 1.5vw + 0.5rem, 1.75rem)` | 600 | 1.3 | `var(--tracking-heading)` = -0.03em | `h3` |

## Color

|------|-------------|-------|

Accent (`#7c5cfc`) is reserved for:
- `.btn-primary` background (primary CTA button: "Book a discovery call")
- Timeline step hover border glow (`box-shadow` with accent)
- Credentials block border-left accent stripe (new, Phase 4)
- `.nav-link.is-active::after` dot (existing)
- Canvas particles (existing, not Phase 4)
- Social link hover state in About section (GitHub, LinkedIn — color transition to `var(--color-accent-hover)`)

Accent is NOT used on: body text, form inputs, section backgrounds, eyebrow labels (those use cyan), credentials card background.

## Interaction States

All interactive elements in Phase 4 must implement these states:

| Element | Default | Hover | Focus-visible | Disabled | Active |
|---------|---------|-------|---------------|---------|--------|
| `.btn-primary` (calendar link, form submit) | Violet bg, white text | `var(--color-accent-hover)` bg, `translateY(-1px)` | Inherited from browser + outline | Opacity 0.6, no pointer | `translateY(0)` |
| `.about-link` | `var(--color-text-secondary)`, bottom border | `var(--color-accent-hover)`, accent border | Focus outline | — | — |
| `.credentials-card` | Surface-2 bg, 3px accent border-left | Spotlight glow via JS | — | — | — |
| `.timeline-step` | Surface-2 bg | `translateY(-5px)`, accent box-shadow | — | — | — |
| Form submit button | `.btn-primary` default | `.btn-primary` hover | Focus ring | `disabled` attr (set by form handler during submit) | — |
| `[data-form-status]` | Empty (min-height 24px) | — | — | — | Shows success (green) or error text |

Form optimistic UI flow:
1. User clicks "Send message"
2. JS sets `submitButton.disabled = true` and `formStatus.textContent = "Sending your request..."`
3. On success: `formStatus.textContent = "Message sent — I'll be in touch within 24 hours."`
4. On error: `formStatus.innerHTML = 'Something went wrong. Email <a href="mailto:...">...</a> directly.'`
5. `submitButton.disabled = false` in `finally` block (already implemented in script.js)

Success message color: `var(--color-success)` (`#4ade80`) — add `color: var(--color-success)` to `.form-status` when it contains success content. Simplest approach: check `formStatus.dataset.state = "success"` and style via `[data-form-status][data-state="success"]` CSS.

## Reveal Animation

All new elements that scroll into view must carry the `reveal` class. The existing `IntersectionObserver` in `script.js` (line 87–99) handles them automatically.

Phase 4 elements requiring `reveal`:
- `.section-heading` wrapper div in process section (already has it)
- Each `article.timeline-step` (5 total, including new step 05)
- `.section-heading` wrapper div in about section
- `.about-bio` div
- `.credentials-card` div
- `.contact-copy` div
- `.contact-form-wrapper` div

iOS Safari: `.reveal` applies `opacity: 0` + `transform: translateY(20px)`. The `.credentials-card` has NO `backdrop-filter`, so combining `reveal` and its card styles is safe.
